New York City Page Prototype

Summary and Takeaways:

This map provides a clear and efficient visual representation of the varying levels of obesity across New York City's boroughs. By using a color-coded scale, it allows viewers to easily discern which areas have higher obesity rates without the need to interpret numerical data. While this map offers a general overview of obesity distribution in NYC, it does not define obesity or explain its health implications. The audience should understand why obesity is a concern and its associated risks.

Vocabulary:

Obesity:
This is a medical condition which is defined as excessive body fat accumulation that poses health risks, characterized by a Body Mass Index (BMI) of 30 or higher. It increases the risk of developing diabetes, heart disease, stroke, and other diseases.

Pipeline for Creating My JSON Dictionary from CSV Data

  1. I first imported the CSV library, the JSON library, and the OS library for checking files.
  2. I then defined the csv_file_path for the CSV source, and the json_file_path for the JSON destination. This makes managing and making adjustments to the files easier.
  3. I opened the CSV file and read the rows into the dictionary using csv.DictReader, which automatically uses the first row headers.
  4. I then organized the data into a nested dictionary by creating sections called "Geography" and "Time Period" for the JSON formatting. Within this, I added the number and percent that are obese within the year and each geographical location.
  5. For writing to JSON, I used json.dump() to ensure the data is secure and to make it more readable by adding indents.
  6. When all of this works, a successful JSON dictionary is made from my clean data, and I can now easily use it for graphs I am planning to make.
